工厂方法模式：

​ 定义：

​ 工厂模式又称工厂方法模式，是一种创建型设计模式，其在父类中提供一个创建对象的方法，允许子类决定实例化对象的类型。

​ 这种设计模式也是 Java 开发中最常⻅的⼀种模式，它的主要意图是定义一个创建对象的接口，让其子类自己决定实例化哪一个⼯厂类，⼯厂模式使其创建过程延迟到子类进⾏。

​ 好处：

​ 用户只需要知道具体工厂的名称就可得到所要的产品，无须知道产品的具体创建过程。

​ 灵活性增强，对于新产品的创建，只需多写一个相应的工厂类。

​ 典型的解耦框架。高层模块只需要知道产品的抽象类，无须关心其他实现类，满足迪米特法则、依赖倒置原则和里氏替换原则。

​ 缺点：

​ 类的个数容易过多，增加复杂度

​ 增加了系统的抽象性和理解难度

​ 抽象产品只能生产一种产品，此弊端可使用抽象工厂模式解决。

​ 模拟场景：

​ 多种类型商品不同接口，统一发奖服务搭建场景

总结：

​ 工厂模式的灵活使用可以让代码逻辑变得更加的清晰，减少ifelse嵌套的使用，让逻辑界限变得清晰合理，同时也避免了创建者与具体的产品逻辑耦合、满足单一职责原则，每一个业务逻辑实现都在所属自己的类中完成，无需更改调用方就可在程序中引入新的产品类型，但是如果使用普通的工厂模式或工厂方法模式，可能会让类的个数过多，增加复杂度，并且每次开发的时候都需要手动增加新的实现类，根据以上代码可以看到，工厂类中的判断还是依赖if…else实现，并没有完全的简化，这个时候就需要进入更深入的思考，垂类业务代码实现后如何自动化添加以及减少工厂类中的ifelse判断，因此需要引入其他的设计模式进行优化。

​ 多思考，根据业务需求灵活运用，不要生搬硬套！！！！！
