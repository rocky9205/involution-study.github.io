正文：

数据结构的学习：

​ 数据结构包括：数组(Array)、栈(stack)、队列(Queue)、链表(LinkList)、散列表(Hash)、堆(Heap)、图(Graph)

​ HashMap作为经典数据结果，也是基础面试必问题目，HashMap是必须深入了解的知识点

​ 其中HashCode的散列是HashMap中很重要的设计思路，所以先学习HahsCode相关知识

HashCode学习：

​ 通过相关面试题深入了解：Why does Java’s hashCode() in String use 31 as a multiplier?

​ 翻译过来就是指：hashCode的计算逻辑中为什么是31作为乘数

​ 当我们点进.hashCode()方法中，可以看到31作为一个固定值写在这个方法中

​ 模拟上述代码验证HashCode值的生成逻辑

通过测试可以得出HashCode的计算时将字符串拆分成char类型的数组然后遍历数组中的每一个元素，设置初始HashCode值为0，获取单字符的ASCII编码值，进行累计叠加的计算，公式：s[0]*31^(n-1) + s[1]*31^(n-2) + ... + s[n-1]

那么问题来了为什么要选择31作为固定乘积值？

网上的热门回答：

  1. 31 是一个奇质数，如果选择偶数会导致乘积运算时数据溢出。
  2. 另外在二进制中，2个5次方是32，那么也就是 31 * i == (i << 5) - i。这主要是说乘积运算可以使用位移提升性能，同时目前的JVM虚拟机也会自动支持此类的优化。
  3. 通过大量的单词计算hashCode，然后使用不同的数字作为乘积，得到碰撞结果，31被使用就很正常了

上诉答案偏理论，会让我们看的云里雾里，我们需要通过实践得出答案，接下来我们使用大量单词并且调整固定乘积值进行hashCode碰撞测试

HashCode验证：

首先我们可以看到HashCode返回值为int类型 int类型占4个字节，每个字节为8位，也就是32位，根据计算机二进制的计算可以得出int类型的范围为-2^31~2^31-1（结尾有二进制整理的内容）

以上就是不同的乘数下的hash碰撞结果图标展示，从这里可以看出如下信息；

  1. 乘数是2时，hash的取值范围比较小，基本是堆积到一个范围内了，后面内容会看到这块的展示。
  2. 乘数是3、5、7、17等，都有较大的碰撞概率
  3. 乘数是31的时候，碰撞的概率已经很小了，基本稳定。

  5. 之前我们学习了int类型的取值范围的来源，计算方式，结果超范围后的结果的展示方式，综上所诉31是最合适的乘积值

Hash值散列分布：

除了以上看到哈希值使用不同乘积值后的一个碰撞概率后，关于散列表也就是hash，还有一个非常重要的点，那就是要尽可能的让数据散列分布。只有这样才能减少hash碰撞，也就是HashMap源码。

总结：

二进制学习（题外话）：

首先我们可以看到HashCode返回值为int类型 int类型占4个字节，每个字节为8位，也就是32位，所以一个整形变量在计算机当中可以用32位二进制表示

又因为二进制里最高位（第一位）为符号为（1：负，0：正），所以剩下31位所以为2^31-1

疑问来了：【最大值：2^31-1】为什么不是2^31？

(1)假设只有三位，其中首位为符号位的话，最大值的情况不就是011吗?也即表示3.
3 = 2^（3-1）-1.

(2)32位的时候同理，01111111 11111111 11111111 11111111 //而这个数表示的值就是2^31-1.
2^31-1 = 2^（32-1）-1.

疑问来了：【最小值：-2^31】为什么不是-（2^31-1）？

(1)因为数据是由二进制补码存储的:
1)当原码为正数的时候，正数的原码反码补码都相同,即00011的反码也为00011，补码也为00011.
2)当原码为负数的时候，反码即按位取反，比如10011为原码,10011可表示-3,那么符号位不变,其余位按位取反即反码11100.那么10011的补码便是11101.

(2)所以可知int所取的最小值原码为：11111111 11111111 11111111 11111111.
它的反码为：10000000 00000000 00000000 00000000. （在原码中表示-0.）
它的补码为：10000000 00000000 00000000 00000001. （在原码中表示-1.）
依次可以类推-2，-3，-4……-2^31！
【推到-2^31】
其原码为10000000 00000000 00000000 00000001
其反码为11111111 11111111 11111111 11111110
其补码为11111111 11111111 11111111 11111111
从上可知最小值只能表示到-2^31，而无法表示到更小的一位：-2^31-1！

4.【综上所诉，int(4字节)范围是：-2^31~2^31-1】！

理解：

​ 二进制转换为十进制:

​ 按权相加法，即将二进制每位上的数乘以权，然后相加之和即是十进制数。

​ 权–》下标

​ 位–》计算的个

​ 负数转换为二进制分为三部：

​ 1、首先将负数转换为对应的原码

​ 2、 再将原码的每一位做取反操作的到反码

​ 3、将反码+1得到补码
